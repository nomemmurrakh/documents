iOS — arm64 + simulatorArm64✅MMKV (via CocoaPods)

One public API across both platforms — it all lives in commonMain. MMKV is bound on Apple targets through the Kotlin CocoaPods plugin, and the library owns MMKV initialization on both platforms, so consumers never lift a finger on that front. There is no JVM/desktop or wasm target in this release.

See Shared KMP Persistence for what this means in practice, and Storage SPI for how the platform- specific binding is isolated behind a minimal internal interface.
