Use Cases

Five shapes of "one typed, reactive document" that come up constantly in real apps — each page sketches a realistic example and links back to the Guide pages that cover the mechanics.

  • Settings & Preferences — the classic SharedPreferences/NSUserDefaults job, typed and observable.
  • Session & User State — the signed-in user, auth tokens, the active profile.
  • Caches & Drafts — last-synced payloads, in-progress form drafts, "continue where you left off."
  • Reactive UI State — anything Compose or SwiftUI should re-render on.
  • Shared KMP Persistence — one storage API, Android and iOS reading the same documents.
